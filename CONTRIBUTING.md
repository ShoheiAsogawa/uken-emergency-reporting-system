# コントリビューションガイド

このプロジェクトへの貢献を歓迎します！以下のガイドラインに従ってください。

## 開発環境のセットアップ

1. リポジトリをフォーク
2. ローカルにクローン
3. 依存関係をインストール: `npm install`
4. 環境変数を設定: `.env`ファイルを作成
5. 開発サーバーを起動: `npm run dev`

## ブランチ戦略

- `main` / `master`: 本番環境（保護ブランチ）
- `develop`: 開発環境
- `feature/*`: 新機能
- `fix/*`: バグ修正
- `docs/*`: ドキュメント更新

## コミットメッセージ

以下の形式に従ってください：

```
<type>(<scope>): <subject>

<body>

<footer>
```

### タイプ

- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメント
- `style`: コードスタイル（フォーマットなど）
- `refactor`: リファクタリング
- `test`: テスト追加/修正
- `chore`: ビルドプロセスやツールの変更

### 例

```
feat(dashboard): 月次損益チャートを追加

月別の損益を可視化するバーチャートを追加しました。
RechartsのBarChartコンポーネントを使用しています。

Closes #123
```

## プルリクエスト

1. 最新の`main`ブランチからブランチを作成
2. 変更をコミット
3. プッシュしてプルリクエストを作成
4. テンプレートに従って記入

### チェックリスト

- [ ] コードがローカルで動作する
- [ ] リンターエラーがない
- [ ] ビルドが成功する
- [ ] 関連するドキュメントを更新
- [ ] テストを追加/更新（該当する場合）

## コードスタイル

- TypeScriptの型を適切に使用
- ESLint/Prettierのルールに従う
- コンポーネントは関数コンポーネントを使用
- カスタムフックでロジックを分離

## テスト

（テストフレームワークを追加する場合）

```bash
npm test
```
